# Durak Imperia Production Readiness

Release is allowed only when:

```bash
GET /api/production/readiness
```

returns `ok=true`.

## Code-Level Status

- App name: `Durak Imperia`
- Android package: `com.durakimperia.game`
- Android target SDK: `35+`
- Google Play Billing: `8+` compatible native purchase flow
- Keywords: `Durak`, `Online`, `Card Game`, `Imperia`
- Ads: Rewarded Video only
- Reward rule: 800 virtual dollars, once every 6 hours
- Real-money betting: not implemented
- Tables use only virtual in-game dollars
- Music is optional
- Baraban, AI helper, voice chat, tournaments, antibot, shop, and admin moderation are implemented in the project

## Required External Release Values

These cannot be generated by code and must be filled in production env:

- `PUBLIC_APP_URL`
- `PRIVACY_POLICY_URL`
- `ADMOB_ANDROID_APP_ID`
- `ADMOB_REWARDED_ANDROID_ID`
- `GOOGLE_SERVICE_ACCOUNT_JSON`
- `ANDROID_RELEASE_KEYSTORE_READY=1`
- `ANDROID_TARGET_SDK=35`
- `GOOGLE_PLAY_BILLING_MAJOR=8`

Set these after real console work:

- `ADMOB_HALAL_CATEGORIES_BLOCKED=1`
- `PREMIUM_PRICES_APPROVED=1`

## Backend Choice

The project uses the Node/Express/Socket.IO backend as the official multiplayer backend. Keep:

```bash
USE_NODE_BACKEND_ONLY=1
```

Set `GDEVELOP_SILVER_ACTIVE=1` only if the project is actually migrated to GDevelop lobby services.

## Manual Play Console Checks

- Block `Gambling & Betting` in AdMob.
- Block `Social Casino Games` in AdMob.
- Create Google Play Billing products listed in `docs/BILLING_PRODUCTS.md`.
- Upload screenshots and store listing from `docs/PLAY_MARKET_RELEASE.md`.
- Publish `web-client/public/privacy.html` or an equivalent hosted privacy policy.
- Use HTTPS for the public app URL.

## Verification

Run:

```bash
npm test
node --check backend/src/routes/production.js
node scripts/release-check.mjs
curl https://YOUR_DOMAIN/health
curl https://YOUR_DOMAIN/api/production/readiness
```
